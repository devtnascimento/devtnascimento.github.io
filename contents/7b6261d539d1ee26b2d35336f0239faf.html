Carteira de Acoes

UMLUseCase
Soy Stocks :: Casos de Usos :: Carteira de Acoes

Description

Fluxo principal - Cotacao

1. Vendedor seleciona opcao "Criar cotacao"
2. Sistema exibe acoes disponiveis
3. Vendedor seleciona acoes a serem cotadas
4. Sistema cadastra cotacao e vincula acoes selecionadas
5. Sistema publica Cotacao
6. Comprador visualza cotacao
7. Comprador seleciona opcao "Realizar Proposta de Compra"
8. comprador define valores da proposta
9. Comprador confirma proposta
10. Sistema  envia proposta ao Vendedor
11. Vendedor visualiza proposta
12. Vendedor aprova proposta

Fluxo excepcional

12.1 Vendedor nao aprova proposta
12.2 Sistema notifica Comprador
12.3 Comprador realiza nova proposta
12.4 Sistema envia nova proposta ao
12.5 Sistema encerra cotacao

Fluxo Alternativo - Configuracao

Agrupar acoes

1. Usuario insere nome do grupo
2. Sistema exibe acoes disponiveis
3. Usuario Seleciona acoes a serem agrupadas
4. Sistema cadastra grupo e vincula acoes selecionadas

Personalizacao

1. Sistema exibe opcoes de visualizacao da cotacao
2. Usuario seleciona opcao desejada
3. Sistema cadastra opcao escolhida`

Fluxo Alternativo - Acao fracionado/lote

1. Sistema exibe opcoes de compra/venda
2. Usuario configura gatilho de compra/venda
3. Sistema monitora o mercado
4. Sistema realiza compra quando acao atender as configuracoes do gatilho
5. Sistema notifica que o usuario esta negociando no mercado fracionario

Fluxo Alternativo - Negocio Rapido

1. Usuario pressiona botao "Negocio Rapido"
2. Sistema exibe opcoes de negocio
3. Usuario seleciona opcao desejada
4. Sistema realiza negocio
